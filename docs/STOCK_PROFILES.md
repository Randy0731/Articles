# 個股專案規則

本文件定義 `Stocks/` 軸。KOL 是來源，Stocks 是研究對象。

## 核心原則

同一篇原文只做一份主整理。

- KOL 文章 / Note / 週報：主整理放 `KOL/<KOL>/`
- 公司新聞、SEC、財報、CEO 訪談、研報若以單一個股為主：主整理可放 `Stocks/<Ticker>/`
- 若 KOL 文章同時對某個已追蹤個股有投資意義，`Stocks/<Ticker>/` 只寫該股相關萃取，不重複貼整篇 KOL 主整理

## Source Of Truth

- `KOL/<KOL>/...`：KOL 原文主整理與完整語境
- `Stocks/<Ticker>/...`：個股投資論述萃取、季度紀錄、里程碑與儀表板
- `Research/...`：非 KOL、非單一個股的產業 / 宏觀 / 橫向研究

若 `KOL/` 主整理與 `Stocks/` 個股萃取出現矛盾，以 `KOL/` 主整理為準，修正 `Stocks/` 內容並記錄到 update log。

## KOL×個股交會等級

| 等級 | 定義 | 動作 |
|---|---|---|
| L0 無關 | 沒有提到 ticker 或研究對象 | 只做 KOL / 來源筆記 |
| L1 提及 | ticker 只在清單、例子、順口提到 | 更新 `indexes/ticker_index.md`，不寫個股筆記 |
| L2 有效觀點 | 對個股有明確分析、風險、催化劑或框架套用 | KOL 主整理 + ticker index；若為已追蹤個股，追加個股摘要筆記 |
| L3 主題個股 | 整篇或主要段落都在分析該股 | KOL 主整理 + 個股筆記 + 個股儀表板 |
| L4 里程碑 | 改變該股投資論述框架，或多母題交會 | KOL 主整理 + 個股里程碑獨立檔 + 個股儀表板 |

判斷句：

> 如果拿掉該 ticker 後，文章主旨仍然成立，多半是 L1/L2；如果拿掉該 ticker 後文章核心不成立，多半是 L3/L4。

## 多 Ticker 分流

一篇多 ticker 文章要逐 ticker 判斷，不整篇套同一等級。

| 情境 | 動作 |
|---|---|
| ticker 只是清單 / 同業比較 / 例子 | 該 ticker = L1，只更新 ticker index |
| ticker 有一段明確分析，但文章主旨不依賴它 | 該 ticker = L2 |
| 文章主體圍繞某 ticker | 該 ticker = L3 |
| 該 ticker 的 thesis 被改寫或出現多母題交會 | 該 ticker = L4 |

若同一篇文章中 **ONDS** 達 L3，但 **SMR** 只是同業清單，只有 **ONDS** 進 `Stocks/ONDS/`，**SMR** 只進 `ticker_index.md`。

## L2 最低門檻

L2 不能只是「有提到」。至少需符合以下任一項：

- 明確立場：看多 / 看空 / 觀望 / 已賣出 / 投機
- 明確風險：增發、現金流、執行、競爭、政策、估值等
- 明確催化劑：財報、股東會、合約、量產、政策、產品節點
- 明確價位或價位性質：目標價、成本價、支撐、期權履約價、事件價位
- 明確框架套用：例如 re-rate、CCC、授權股稀釋、EPS 裁判、機構洗盤
- 對既有 thesis 有增量資訊：補強、削弱、轉折或矛盾

若只是新聞標題、分析師目標價、群友喊價、或 KOL 順口列名，維持 L1。

## 寫入規則

### L1

- 只更新 `indexes/ticker_index.md`
- 不寫 `Stocks/<Ticker>/`
- 不更新儀表板

### L2

- 原文主整理放 KOL 或來源資料夾
- `Stocks/<Ticker>/notes/` 或當季檔追加摘要型個股筆記
- 更新 ticker index
- 若該股有儀表板，只增量更新相關欄位；不一定改「當前立場快照」

### L3

- 原文主整理放 KOL 或來源資料夾
- `Stocks/<Ticker>/` 建立較完整個股筆記
- 更新該股 index 與儀表板；可更新立場、風險、催化劑與持倉依據

### L4

- 原文主整理放 KOL 或來源資料夾
- `Stocks/<Ticker>/milestones/` 建立里程碑檔
- 更新該股 index 與儀表板；必須檢查並更新立場快照、關鍵風險、持倉依據

## 儀表板更新範圍

| 等級 | 儀表板更新 |
|---|---|
| L1 | 不更新 |
| L2 | 增量更新價位、風險、催化劑或 open question；通常不改綜合立場 |
| L3 | 更新相關欄位，可調整綜合立場或持倉判斷依據 |
| L4 | 必須更新綜合立場、關鍵風險、持倉判斷依據與里程碑連結 |

儀表板是當前狀態摘要，不是全文筆記。不要把所有細節塞進儀表板，細節留在季度檔、長文或里程碑檔。

### 催化劑狀態維護

每次入庫 L2 以上個股筆記，或查詢「接下來看什麼」「催化劑」時，檢查相關個股儀表板與 `indexes/catalyst_index.md`：

| 狀態 | 定義 | 動作 |
|---|---|---|
| 待追蹤 | 事件日期尚未到 | 留在近 30 天催化劑或 catalyst index |
| 待確認 | 日期已到，但專案內沒有結果資料 | 標記待確認，不推測結果 |
| 已兌現 | 事件發生且方向符合原追蹤重點 | 移到已兌現催化劑並補結果 |
| 未兌現 | 事件未發生或結果明確落空 | 記錄落空原因與來源 |
| 延期 | 事件推遲到新日期或時間未定 | 更新日期或標待確認 |

儀表板保留近 30 天待追蹤與近 90 天已兌現事件；完整歷史保留在 `indexes/catalyst_index.md` 與原筆記。

## 季度歸檔規則

季度檔只用於一般筆記。L4 里程碑進 `milestones/`，長文進 `longform/`，不受季度合併檔限制。

### 季度判斷優先順序

1. **發文時間**：若原文有明確發文日期，用發文日期判斷季度。
2. **事件日期**：若文章沒有發文日期，但明確是某事件的官方公告 / 財報 / 合約，以事件日期判斷季度。
3. **整理日期**：若發文日期與事件日期都不明，用整理日期判斷季度，並在筆記中標「發文時間：推估 / 未確認」。

不要用「催化劑未來發生日」決定季度。未來事件寫進儀表板或 catalyst index，但筆記歸檔仍依發文時間 / 事件日期 / 整理日期。

### 季度換算

| 月份 | 季度 |
|---|---|
| 1-3 月 | Q1 |
| 4-6 月 | Q2 |
| 7-9 月 | Q3 |
| 10-12 月 | Q4 |

檔名格式：

```text
<Ticker>_筆記_YYYYQ<N>.md
```

例如：

- `ONDS_筆記_2026Q2.md`
- `EOSE_筆記_2026Q2.md`
- `SMR_筆記_2026Q2.md`

### 自動建立新季度檔

若判斷出的季度檔不存在，自動建立：

```md
# <Ticker> 筆記 YYYYQ<N>

此檔用於收錄 YYYYQ<N> 的一般 <Ticker> 筆記。里程碑事件放 `Stocks/<Ticker>/milestones/`，長文深度分析放 `Stocks/<Ticker>/longform/`。

---
```

### 補舊文

若整理日期是本季，但原文發文時間屬於過去季度：

- 寫入原文所屬季度檔
- 不改寫當季檔
- 在筆記基本資訊中保留整理日期
- update log 記錄「補入舊文」

### 跨季文章

若文章回顧上一季、展望下一季，但發文在本季：

- 寫入發文日期所屬季度
- 相關事件日期寫入 catalyst index / dashboard
- 不因內容涵蓋多季而拆成多個季度檔

### 日期不確定

若只能推估月份：

- 可推估季度時，寫入該季度，並標「推估」
- 無法推估月份時，用整理日期季度，並標「發文時間：未確認」

## 新 ticker 偵測與升級流程

所有新文章都要抽取 ticker，但不是每個 ticker 都建立個股專案。

### ticker 狀態

| 狀態 | 定義 | 動作 |
|---|---|---|
| 已追蹤個股 | 已存在於 `stock_projects` 或 `Stocks/<Ticker>/` | 依 L0-L4 規則處理 |
| 索引中 ticker | 只是在文章中出現，尚未成為研究對象 | 更新 `indexes/ticker_index.md` |
| 候選追蹤個股 | 出現多次 L2 或單篇 L3/L4，但尚未正式追蹤 | 更新 `indexes/watchlist_index.md` |
| 正式追蹤個股 | 使用者指定或符合升級條件 | 建立 `Stocks/<Ticker>/` 專案 |

### 預設不升級

以下情況不建立 `Stocks/<Ticker>/`：

- ticker 只在清單、表格、同業比較中出現
- 只是分析師目標價或新聞標題提及
- KOL 只是用來舉例，沒有明確個股 thesis
- 單次 L2，但沒有後續催化劑或持續追蹤價值

### 升級條件

符合任一條件，可以建立個股專案：

- 使用者明確要求追蹤
- 單篇達 L3 / L4，文章核心就是該 ticker
- 同一 ticker 在多篇文章中反覆達 L2 以上
- 出現明確催化劑、風險、價位、框架與後續追蹤需求
- 已被納入使用者持續研究清單

### 升級動作

建立：

```text
Stocks/<Ticker>/
├── raw/
├── notes/
├── quarterly/
├── milestones/
├── longform/
├── <Ticker>_儀表板.md
└── <Ticker>_index.md
```

並更新：

- `kol_config.yaml` 的 `stock_projects`
- `indexes/ticker_index.md`
- `indexes/watchlist_index.md`（若原本在 watchlist，標記已升級）
- `logs/update_log.md`

### 不確定時

如果 ticker 看起來值得觀察，但升級理由不夠強，先進 `indexes/watchlist_index.md`。不要硬建完整個股專案。

### Watchlist 冷卻與升級

- 候選 ticker 連續 90 天沒有新增 L2 以上資料，不刪除，只在 `watchlist_index.md` 標記「冷卻」
- 使用者明確指定即可從 watchlist 升級為正式追蹤
- 已升級的 `Stocks/<Ticker>/` 不自動降級；若不再追蹤，改標記「封存」而非刪除

## ONDS 專案

### 資料夾

```text
Stocks/ONDS/
├── raw/
├── notes/
├── quarterly/
├── milestones/
├── longform/
├── ONDS_儀表板.md
└── ONDS_index.md
```

### 檔案類型

| 類型 | 檔名 | 收錄條件 |
|---|---|---|
| 季度合併檔 | `ONDS_筆記_YYYYQN.md` | 一般訂單、合約、財報解讀、社群評論、機構動向、技術面分析 |
| 里程碑獨立檔 | `ONDS_里程碑_文章標題.md` | 改變 ONDS 論述框架，或多母題交會點 |
| 長文獨立檔 | `ONDS_文章標題.md` | 整理後 > 200 行，且不屬里程碑等級 |

判斷流程：

1. 是否符合 L4 里程碑？是則進 `milestones/`
2. 否，整理後是否 > 200 行？是則進 `longform/`
3. 否，進當季合併檔

季度切分時點：每年 1 / 4 / 7 / 10 月第一篇新筆記時建立新季度檔。

### ONDS 母題標籤

固定使用以下標籤，避免新造同義詞：

- `#十年繁榮` — CEO 定調的 10-year boom 敘事
- `#142K據點TAM` — 由下而上的 TAM 估算
- `#空地一體` — 空中 + 反制 + 地面整合的戰略框架
- `#PS小於1倍` — 併購套利財務模板
- `#DualUse` — 軍事 / 國土安全 / 公共安全 / 基建四客群通吃
- `#以色列併購` — 以色列公司收購案
- `#以色列訂單` — 以色列國防部 / IDF 相關訂單與合約
- `#以色列人事` — 前 Rafael 高層、IDF 將領等以色列人才招募
- `#機構洗盤` — PLTR 類比的洗盤劇本
- `#EPS終極裁判` — 併購成敗以 EPS 提升為核心標準
- `#授權股稀釋` — 授權股、增發與稀釋追蹤
- `#NATO佈局` — 歐洲與北約市場拓展
- `#垂直整合` — SPAI / LPTH / KOPN 等技術堆疊投資

舊標籤不回頭重打。新筆記一律使用新標籤。

## 通用個股筆記固定格式

```md
## 文章標題

- **來源**：
- **類型**：
- **發文時間**：
- **整理日期**：
- **source_id**：
- **raw 路徑 / URL**：
- **OCR 狀態**：
- **母題標籤**：
- **事件類型**：
- **原文主整理**：

### 主旨

### TICKER 觀點摘要

| 面向 | 內容 |
|---|---|
| **立場** | 看多 / 看空 / 觀望 / 已賣出 |
| **關鍵價位** | 買入價、目標價、停損價等 |
| **技術面** | 原文有提才寫 |
| **基本面** | 原文有提才寫 |
| **催化劑 / 風險** | 原文有提才寫 |

未提及的列直接刪除，不寫「未提及」。

### 留言區關鍵訊號

### 關鍵金句

### 核心概念

### 呼應連結

### 本篇在 TICKER 投資論述中的歷史定位
```

「本篇在 TICKER 投資論述中的歷史定位」只在 L4 里程碑使用。寫了此段就應該獨立成里程碑檔。

### 儀表板更新

每篇 L2 以上正式追蹤個股新筆記寫完後，更新對應 `Stocks/<Ticker>/<Ticker>_儀表板.md`。

更新方式：

1. 讀儀表板現況
2. 依新筆記增量更新欄位，不整份重寫
3. 更新頂部「最後更新日期」與「觸發來源」
4. 已過期催化劑從近 30 天移到已兌現
5. 已過 90 天的兌現催化劑從儀表板移除

### 最終檢查

- 新筆記是否歸入正確檔案類型
- 觀點摘要表格是否刪除未提及列
- 關鍵金句是否純引用
- 呼應連結是否 <= 3 條
- 對應個股儀表板是否同步更新
