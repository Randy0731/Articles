# 寫作格式規格

本文件是格式硬規格。範例模板見 [docs/TEMPLATES.md](docs/TEMPLATES.md)。

## 1. 模板選擇

| 內容類型 | 主整理位置 | 使用格式 |
|---|---|---|
| 一般 KOL 短 Note | `KOL/<KOL>/notes/<主筆記>.md` | 通用短文格式 |
| 一般 KOL 長文 | `KOL/<KOL>/articles/` | 通用文章格式 |
| Bytc 短文 | `KOL/Bytc/notes/bytc筆記.md` | Bytc 短文格式 |
| 大叔文章 / Note | `KOL/大叔美股筆記/notes/` 或 `articles/` | 大叔文章 / Note 格式 |
| KP 週報 | `KOL/KP_FOMOSoc/weekly/` | KP 週報格式 |
| KP 深度分析 | `KOL/KP_FOMOSoc/deep_dive/` | KP 框架拆解格式 |
| KP 純框架 / 個股文 | `KOL/KP_FOMOSoc/articles/` | KP 框架 / 個股格式 |
| 宋分文章 | `KOL/宋分/articles/` | 宋分框架格式 |
| 美股老司機文章 | `KOL/美股老司機/notes/美股老司機投資筆記.md` | 編號文章格式 |
| 已追蹤個股 L2 | `Stocks/<Ticker>/quarterly/` 或 `notes/` | 個股摘要型格式 |
| 已追蹤個股 L3 | `Stocks/<Ticker>/quarterly/` 或 `longform/` | 個股完整筆記格式 |
| 已追蹤個股 L4 | `Stocks/<Ticker>/milestones/` | 里程碑格式 |
| 非 KOL 產業 / 宏觀研究 | `Research/industry/` 或 `Research/macro/` | Research 格式 |
| 跨 KOL 比較筆記 | `Research/comparisons/` | 比較筆記格式 |

## 2. 全域 Markdown 規格

- 使用繁體中文。
- ticker 一律粗體大寫：`**ONDS**`、`**EOSE**`、`**SMR**`。
- 價位一律用反引號：`` `$12.50` ``。
- 日期一律 `YYYY-MM-DD`；不確定時寫「約 YYYY 年 M 月（推估）」。
- 每個主要區塊之間空一行。
- 合併筆記中每篇文章結尾加 `---`。
- 合併筆記內文採 append-only；檔案頂部文章索引依發文時間倒序維護。
- 沒有內容的可選區塊直接省略，不寫「無」「未提及」，除非該模板明確要求保留。
- 個股摘要表中未提及的列直接刪除；一般觀點表可填「未說明」。
- 引文只保留真正有證據價值的短引文。
- 整理者推論必須放在「整理者延伸」或明確標示「整理者判斷」。
- 每篇入庫筆記必須有 `source_id`；實體檔案需記錄 raw 路徑或原始路徑；OCR / PDF / 截圖需記錄 OCR 狀態。

## 2.5 表格優先規則

以下資訊優先用表格，除非只有 1 個短項目：

| 資訊類型 | 使用表格 |
|---|---|
| 多 ticker 分流 | Ticker 分流表 |
| 個股觀點 | 個股觀點表 |
| 價位、成本、目標價、支撐 | 價位表 |
| 催化劑、財報日、股東會、合約日期 | 催化劑表 |
| 風險清單 | 風險表 |
| 框架摘要 | 框架摘要表 |
| 跨 KOL 比較 | KOL 比較表 |
| 週報主題 | 主題索引表 |
| 儀表板更新 | 儀表板既有表格 |

段落文字保留給主旨、投資論述解讀、觀點轉折、整理者延伸。

## 2.6 標準表格

### Ticker 分流表

多 ticker 文章必填。

```md
| Ticker | 文章角色 | 交會等級 | 動作 | 理由 |
|---|---|---|---|---|
| **TICKER** | 主角 / 重要案例 / 同業比較 / 順口提及 | L1 / L2 / L3 / L4 | ticker index / stock note / milestone / watchlist | ... |
```

### 價位表

同文出現 2 個以上價位時使用。

```md
| Ticker | 價位 | 價位性質 | 來源 / 說話者 | 時間 | 備註 |
|---|---|---|---|---|---|
| **TICKER** | `$XX` | 目標價 / 成本價 / 希望價 / 事件價位 / 分析師目標價 / 期權履約價 | ... | YYYY-MM-DD | ... |
```

### 催化劑表

```md
| 日期 | 事件 | 相關 ticker | 影響方向 | 追蹤重點 | 出處 |
|---|---|---|---|---|---|
| YYYY-MM-DD | ... | **TICKER** | 正面 / 負面 / 中性 / 未定 | ... | ... |
```

### 風險表

```md
| 風險 | 相關 ticker | 風險等級 | 觸發條件 | 觀察指標 | 出處 |
|---|---|---|---|---|---|
| ... | **TICKER** | 高 / 中 / 低 | ... | ... | ... |
```

### 框架摘要表

若同文拆出 2 個以上框架，先用摘要表，再逐一展開。

```md
| 框架 | 核心邏輯 | 適用情境 | 觀察指標 | 相關 ticker |
|---|---|---|---|---|
| ... | ... | ... | ... | **TICKER** |
```

### 跨 KOL 比較表

```md
| KOL | 觀點 | 訊號強度 | 時間框架 | 主要證據 | 風險 / 反例 |
|---|---|---|---|---|---|
```

### 合併筆記文章索引表

所有 KOL 主筆記與個股季度合併檔建議放在檔案頂部。表格排序依 `發文時間` 倒序；內文仍 append-only。

```md
| 發文時間 | 整理日期 | 標題 | source_id | 相關 ticker | 主題 tags | 備註 |
|---|---|---|---|---|---|---|
```

## 3. 通用文章格式

適用於大部分 KOL 長文、研究文章、非特化文章。

必填：

```md
## [文章標題]

- **KOL**：
- **來源**：
- **類型**：
- **發文時間**：
- **整理日期**：
- **原始檔案 / URL**：
- **source_id**：
- **raw 路徑 / URL**：
- **OCR 狀態**：
- **相關 ticker**：
- **主題 tags**：

### 主旨
```

可選區塊依順序出現：

```md
### Ticker 分流
### 結構化分析
### 個股觀點
### 價位與催化劑
### 核心框架 / 心法
### 關鍵證據
### 風險與反例
### 延伸追蹤
### 整理者延伸
### 修訂紀錄
```

規則：

- `### 主旨`：2-4 句，說明核心問題、主要結論與研究價值。
- `### Ticker 分流`：多 ticker 文章必填，用 Ticker 分流表。
- `### 個股觀點`：有 ticker 且有觀點才寫。
- `### 價位與催化劑`：同文有多個價位或事件時用表格。
- `### 核心框架 / 心法`：最多 3 點。
- `### 延伸追蹤`：只放明確日期或事件。
- `### 修訂紀錄`：只在補充版、修正版、OCR 改善或重大資料更正時使用。

## 5. 個股筆記格式

適用於 `Stocks/<Ticker>/`。

### L2 摘要型

```md
## [來源 KOL] 對 **TICKER** 的觀點摘要：[自擬標題]

- **來源 KOL**：
- **原文主整理**：
- **原始來源**：
- **source_id**：
- **raw 路徑 / URL**：
- **OCR 狀態**：
- **類型**：
- **發文時間**：
- **整理日期**：
- **交會等級**：L2
- **事件類型**：
- **主題 / 母題標籤**：

### 對 **TICKER** 的影響

### 觀點摘要

| 面向 | 內容 |
|---|---|
| **立場** |  |
| **關鍵價位** |  |
| **技術面** |  |
| **基本面** |  |
| **催化劑 / 風險** |  |

### 價位與催化劑

| 類型 | 內容 | 日期 / 價位 | 來源 | 備註 |
|---|---|---|---|---|

### 風險表

| 風險 | 風險等級 | 觸發條件 | 觀察指標 | 出處 |
|---|---|---|---|---|

### 關鍵證據

### 核心概念

### 呼應連結

---
```

規則：

- `觀點摘要` 未提及的列直接刪除。
- `呼應連結` 最多 3 條。
- 不重貼 KOL 主整理全文。

### L3 完整個股筆記

使用 L2 格式，但 `交會等級` 寫 L3，且可增加：

```md
### 論述變化
### 可追蹤事件
### 風險清單
```

### L4 里程碑

使用 L3 格式，且必須加：

```md
### 本篇在 **TICKER** 投資論述中的歷史定位
```

只要寫此段，就必須放 `milestones/`，不可放季度檔。

更新規則：

- L2：只更新相關欄位，不硬改綜合立場。
- L3：可調整綜合立場、風險與催化劑。
- L4：必須檢查立場快照、關鍵風險、持倉依據。

## 9. 省略規則

可以省略：

- 沒有個股時省略個股觀點表
- 沒有金句時省略關鍵金句
- 沒有明確事件時省略延伸追蹤
- 沒有留言互動時省略留言區關鍵訊號
- 沒有框架時省略框架拆解，或改寫為「套用框架」

不可省略：

- 基本資訊
- 主旨
- 來源 / 原始檔案或 URL
- 整理日期
- 相關 ticker / 主題 tags
- 交會等級（個股筆記）
