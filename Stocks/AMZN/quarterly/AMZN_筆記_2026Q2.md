### 對 **AMZN** 的影響

KP 將 Amazon 放在 big-tech AI capex 「prove-it stage」裡比較，並把 **AMZN** 寫成 cloud leader 轉向 AI infrastructure supplier 的候選。AWS Q1 revenue `$37.6B`、+28%，創 15 個季度最佳增速，annualized run-rate 約 `$150B`；operating profit `$14.2B`，margin 37.7%。KP 的重點是 AI workload 會帶動 S3、database、networking、security 與 compute architecture 等 core AWS services，而不只是單次 GPU / accelerator rental。

本文延續第37期 Jassy shareholder letter thesis：Graviton / Trainium / Nitro 自研晶片業務年化 >`$20B`，q/q +40%；KP 口徑稱 OpenAI 2027 起在 AWS 啟用約 2GW Trainium capacity，Anthropic current / future Trainium 最高 5GW 並有超過 `$100B` 十年 AWS spend，Meta 也承諾大量 Graviton cores 用於 agent AI workload。這些合約與 workload 細節先按 KP 口徑入庫，待後續 official filing / transcript 校準。

KP 口徑 AWS backlog / RPO 從前季 `$244B` 升至 `$364B`，且不含近期 Anthropic >`$100B` deal；他將 2026 約 `$200B` capex 解讀為 2027-2028 revenue assets。對 **AMZN** 的 L3 增量是「capex 有收據」更進一步，但真正檢查仍是 Trainium performance、RPO conversion、AWS margin、FCF、depreciation 與 self-designed silicon 是否能外部化。

KP 將 Amazon 收購 Globalstar 解讀為 **AMZN** AI / satellite infrastructure thesis 的 L3 補強：Amazon 不是單純買一家公司，而是買 MSS spectrum、全球落地許可、既有 Apple iPhone Emergency SOS 生態與 direct-to-device 的時間差。這呼應宋分 2026-04-21 / 04-25 對 Amazon Leo / Globalstar 的 coverage + control 框架，也把 **AMZN** 的 Cycle 3 從 AWS AI compute 延伸到 connectivity layer。

### 對 **AMZN** 的影響

KP 將 Andy Jassy 2026-04-09 年度股東信解讀為 Amazon 對 2026 年約 2000 億美元 capex 疑慮的正式答辯。這篇對 **AMZN** 是 L3 補強：Jassy 用 AWS AI annualized revenue、Trainium / Graviton / Nitro 自研晶片 revenue、OpenAI 大型合約與未來對外銷售整機櫃晶片 optionality，說明 capex 背後有需求、合約與產能預售，而不是單純盲目燒錢。

這與宋分 2026-04-21 / 04-22 的 AMZN capital cycle、ROIC x Organic Growth 框架同方向：短期自由現金流和折舊壓力不能單獨判斷，重點是新增資本是否轉成 AWS AI revenue、custom silicon margin、RPO conversion 與更高 ROIC。風險仍在於合約品質、客戶集中、Trainium utilization、外售晶片是否真的高毛利，以及 2000 億美元 capex 是否造成 depreciation / FCF 壓力。

| 面向 | 內容 |

### 對 **AMZN** 的影響

Bytc 在四巨頭財報檢查後補上 capex 估計更新：Amazon 2026 capex 仍維持約 `$200B`，但四巨頭合計從約 `$665B` 上修至約 `$710B`。對 **AMZN** 的增量是「Amazon 不是孤立高 capex」，而是 hyperscaler 整體仍在擴大 AI / cloud infrastructure；這補強 AWS demand receipt / capital cycle thesis，但不是新的單股買賣點。

Amazon official Q4 2025 release 已校準 Andy Jassy 對 2026 約 `$200B` capex 的表述，且重點放在 AWS、AI、chips、robotics 與 low earth orbit satellites。這篇 Bytc Note 沒有改寫 AMZN thesis，主要是把 `$200B` 放進四巨頭共同上修的背景中：若 Microsoft、Google、Meta 也加碼，AI infrastructure 的競爭與需求都更激烈。

### 觀點摘要

| 面向 | 內容 |
|---|---|
| **立場** | 偏正面 L2；Bytc 將四巨頭 capex 上修視為 AI 基建需求仍強，但 **AMZN** 自身 capex 數字本篇是維持而非再上修 |
| **關鍵數字** | Amazon 2026 capex `$200B`；四巨頭合計 Bytc 口徑約 `$710B`，原估約 `$665B` |
| **基本面含義** | AMZN `$200B` capex 更像 hyperscaler 軍備競賽的一部分，需要用 AWS revenue / margin / RPO conversion / FCF 驗證回收 |
| **驗證點** | AWS revenue、AWS operating margin、capex、depreciation、FCF、Trainium / Bedrock demand、power / component cost |
| **主要風險** | 四巨頭同步加碼可能意味 demand 強，也可能意味競爭者同時擴產、壓低未來 pricing power；高 capex 仍可能壓縮 AMZN FCF |
