## KP / FOMOSoc：AWS 再加速與 Project Rainier，證明還在 AI 雲端牌桌上

- **source_id**：`KP_FOMOSoc-20251101-kp-thinking-note-14-meta-cloud-nvo-fiserv-cmg-nvidia-e81ba8c3`
- **來源**：KP / FOMOSoc Substack 週報
- **原文**：https://www.fomosoc.com/p/metanvidiaai-kp14
- **發文時間**：2025-11-01 02:45 UTC（台北 10:45）
- **整理日期**：2026-04-29
- **交會等級**：L2（已追蹤個股歷史補強）
- **主題 tags**：#雲端基建 #AI基建 #財報 #ReRating #競爭風險

### 主旨

KP 在第 14 期將 AWS Q3 財報放進雲端三國比較：市場原本擔心 AWS 在 AI 浪潮中慢了一拍，但 AWS revenue +`20.2%`、annualized revenue 約 `$132B`、operating profit 約 `$10.2B`、margin 約 `31%`，顯示 AWS 沒有掉出牌桌。Project Rainier 與約 100 萬顆 Trainium chips for Anthropic，則是 AWS 用自研 AI chip 與客戶綁定回應 Azure / Google Cloud 壓力。

### 個股觀點

| 面向 | KP 觀點 | 對 **AMZN** 的資料庫含義 |
|---|---|---|
| AWS revenue reacceleration | +`20.2%` 是 2022 年以來首次顯著再加速，市場反應更像鬆一口氣 | 補強 AWS 最壞成長放慢期可能已過，但仍需用後續 RPO / revenue conversion 驗證 |
| Scale advantage | Andy Jassy 強調在 `$132B` annualized revenue 上成長 20%，絕對新增收入仍很大 | 對照宋分 AMZN capital cycle thesis：規模仍是雲端現金流底座 |
| Profit stabilizer | AWS operating profit 約 `$10.2B`，margin 約 `31%` | 追 AWS 是否能在 AI capex 高峰維持 margin / FCF |
| Project Rainier | AWS 為 Anthropic 建設 AI data center，提供約 100 萬顆 Trainium chips | 與第 13 期 Anthropic / Google TPU 分工一起看：training / inference 分流下，AWS 需要證明 Trainium workload 黏性 |

### 延伸追蹤

| 事件 | 追蹤重點 |
|---|---|
| 每季 AWS earnings | revenue growth、operating margin、absolute dollar growth、RPO / backlog conversion、AI workload revenue |
| Project Rainier / Anthropic updates | Trainium utilization、training workload share、delivery schedule、capex / depreciation、customer concentration |
| 三大雲端比較 | Azure AI growth、Google Cloud TPU / backlog、AWS price / capacity / custom-chip differentiation |

### 風險與限制

- KP 的判斷是「AWS 沒有掉隊」，不是說 AWS 已重新取得 AI 雲端領先權；Azure 與 Google Cloud 在 AI 客戶與增長上仍有更強敘事。
- Project Rainier 需要看 revenue recognition、utilization、Trainium performance 和 Anthropic workload split，不能只因建設規模大就直接寫成高 ROI。
- 本文不是買賣建議。

## KP / FOMOSoc：AWS 大當機與 Anthropic 分工，不等於 AWS thesis 崩壞

- **source_id**：`KP_FOMOSoc-20251025-kp-thinking-note-13-openai-gm-quantum-aws-tsla-intc-nflx-bd0ee08c`
- **來源**：KP / FOMOSoc Substack 週報
- **原文**：https://www.fomosoc.com/p/altmanaws-kp13
- **發文時間**：2025-10-25 02:30 UTC（台北 10:30）
- **整理日期**：2026-04-29
- **交會等級**：L2（已追蹤個股歷史補強）
- **主題 tags**：#雲端基建 #AI基建 #競爭風險 #執行風險 #ReRating

### 主旨

KP 將 2025-10-20 AWS outage 解讀為 DNS / IAM 層級故障，而非整座 AWS 基礎設施失效；市場冷靜，反映 AWS 的高轉換成本與 critical infrastructure 地位。同篇對 Anthropic 轉向 Google TPU 的解讀也不是 AWS AI 佈局崩壞，而是頂級 AI 公司把 training、inference、latency、cost 與 bargaining power 拆給不同雲端與晶片。

### 個股觀點

| 面向 | KP 觀點 | 對 **AMZN** 的資料庫含義 |
|---|---|---|
| AWS outage | AWS 問題像城市「地址查詢系統」失效，服務本身仍健康，但 IAM / DNS 失靈造成授權連鎖故障 | 補強 AWS moat 的同時，提高 outage frequency、SLA credits、multi-region adoption 與監管檢查權重 |
| 客戶黏性 | 遷移數位基礎設施成本極高，客戶即使痛苦也難以輕易離開 AWS | 延續既有 **AMZN** thesis 中 AWS backlog / margin / switching cost 檢查 |
| 災難財 | Outage 可能推動企業購買更昂貴的多區域備援與 resilience solution | 可加入 AWS revenue mix / margin 的正反兩面觀察 |
| Anthropic / Google TPU | Google TPU deal 是 Anthropic 的 inference / scale 分工，不代表 AWS 失去 training 核心 | 後續需追 AWS Trainium、Anthropic workload split、cloud AI revenue 與 customer concentration |

### 延伸追蹤

| 事件 | 追蹤重點 |
|---|---|
| 每次 AWS outage / postmortem | outage scope、SLA credits、enterprise churn、multi-region adoption、regulatory response、critical infrastructure rules |
| 每季 AWS 財報 | AWS revenue、operating margin、backlog / RPO conversion、resilience products、AI infrastructure capex ROI |
| Anthropic cloud spend update | AWS / Trainium training usage、Google TPU inference usage、latency、cost per token、Anthropic customer growth |

### 風險與限制

- KP 不是在說 outage 對 AWS 無害，而是說市場暫時把它定價為可控事故；若事件重複或客戶架構真的多雲化，AWS moat 仍會被重估。
- Anthropic 與 Google deal 對 AWS 是公關挫敗，仍需用 training workload、Trainium adoption 與 cloud revenue 驗證是否只是分工，或真的代表 AWS AI competitive position 弱化。
- 本文不是買賣建議。
