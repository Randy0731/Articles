## 主旨

Bytc 的核心判斷不是「Google Search 會立刻被 AI 取代」，而是搜尋這台高利潤印鈔機仍然存在，但 AI Overviews / AI Mode / Gemini / Agent workflow 會讓印鈔機變得更昂貴，並迫使廣告 auction、流量入口、轉換歸因與 margin 結構重新定價。

傳統搜尋的價值在於使用者帶著明確意圖進場，廣告商願意為這種 intent capture 付費。AI 搜尋改變的是路徑：使用者不一定再看十個藍色連結，而是先看到答案、比較、摘要，甚至由 agent 直接執行任務。對 Google 而言，問題不只是流量是否還在 Google，而是 AI answer / agent 層能否維持或提高廣告價格、ad load、轉換率與 margin。

## 結構化分析

| 主題 | Bytc 觀點整理 | 投資含義 |
|---|---|---|
| 搜尋仍在印鈔 | 使用者搜尋時帶著明確意圖，這是 Google Search 長期高利潤的根源 | **GOOG** / **GOOGL** 的核心 moat 仍在，不是立即崩壞 |
| AI answer layer | AI Overviews 把搜尋從連結列表改成摘要答案，可能降低部分外部點擊 | 舊式 CPC / link auction 需要被 answer ads、context ads 或 task-completion ads 補上 |
| Agent action layer | agent 若直接比較、下單、預訂或執行任務，搜尋漏斗會從「搜尋 -> 點擊 -> 商家」變成「提問 -> 比較 -> 執行」 | Google 若不能成為 agent broker，intent capture 的 economics 會被重寫 |
| Capex / depreciation | AI search 的每次 query 有更高 compute cost，且 data center / TPU / GPU / power capex 推高折舊 | 風險從 revenue collapse 轉成 margin / FCF / multiple compression |
| Ad auction 重建 | 傳統 keyword CPC / CPA 可能轉為 multimodal、contextual、task、conversion、agent-intervention 型廣告 | 若 Google 成功改造 auction，Search 可升級；若失敗，spread 會被壓縮 |
| 分發護城河 | Chrome、Android、YouTube、Maps、Gmail、Gemini、Cloud 與自研 TPU 仍是強分發和成本控制資產 | Google 有條件把 AI 嵌入既有入口，但每多一層 AI 都要付 compute 成本 |
| 競爭壓力 | OpenAI、Perplexity、Microsoft、Meta、TikTok / ByteDance、Amazon vertical search 都在搶入口或 intent | 競爭不是單點替代，而是入口被多個場景分流 |
| 多空分歧 | 多頭看 Google 能把 AI 變成更高價值 ad broker；空頭看 AI 先吃掉 paid clicks 與 margin | 本篇更偏「重新承保」而非「直接歸零」 |

## 核心框架 / 心法

- AI 不一定摧毀搜尋，但會把搜尋從低成本連結分發改成高成本答案與任務完成。
- Google 的 legacy profit pool 同時是 moat 與包袱；越賺錢的舊模式，越怕新模式先壓 margin。
- Search moat 的再定價重點是 ad load、conversion、pricing、attribution 與 compute cost，而不是只問使用者會不會繼續用 Google。
- AI search 競爭要同時看分發入口、模型能力、inference 成本、廣告格式與 agent action 權限。

## 風險與反例

| 風險 / 反例 | 對應 ticker | 判讀 |
|---|---|---|
| AI Overviews / AI Mode cannibalize paid clicks | **GOOG**, **GOOGL** | 若 answer page 減少外部點擊且廣告格式補不上，Search monetization 會受壓 |
| AI query compute cost 推高折舊與 capex | **GOOG**, **GOOGL**, **MSFT**, **META** | 即使 revenue 成長，margin / FCF multiple 仍可能被壓 |
| Google 成功把 AI answer / agent 變成新 ad broker | **GOOG**, **GOOGL** | 這是多頭反例；若 conversion 更好，AI 可能提高 ad pricing |
| OpenAI / Perplexity / Microsoft / Amazon / TikTok 分流入口 | **GOOG**, **GOOGL**, **MSFT**, **AMZN** | 競爭不是單一搜尋框，而是不同高意圖場景被切走 |
