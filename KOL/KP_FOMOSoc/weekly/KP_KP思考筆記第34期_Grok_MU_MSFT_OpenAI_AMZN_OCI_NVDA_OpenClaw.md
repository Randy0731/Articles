## 0. 投資圖書館：把週報當成認知複利系統

KP 開場說明 FOMO 研究院的定位不是一般週更新聞，而是一座持續擴充的投資圖書館。深度分析的任務是拆商業模式、Bull / Bear case、產業底層邏輯；週末筆記的任務則是追蹤新事件，驗證或修正先前框架。

整理者判定：這段補強「投資圖書館 / 認知複利框架」。對本資料庫也很重要：補舊文不是重寫歷史，而是讓同一 KOL 的底層框架可被跨期查詢與追蹤。

## 1. Grok 金融導師：AI 從資料量競爭轉向專家思維萃取

KP 將 xAI 招募銀行家、信貸專家與交易員解讀為「專業大腦移植」：通用模型靠大量資料訓練，在金融這種容錯率極低、隱性判斷很多的領域仍容易停在平庸摘要；xAI 轉而讓專家把交易、信貸、LBO、CLO 定價等推理步驟轉成模型可學的思考流程。

這段的投資含義不是 xAI 單股，因為 xAI 未上市，而是 AI 競爭從「誰有最多資料」轉向「誰能把高品質人類專業知識萃取進模型」。若 Grok 結合 X 即時資訊與金融推理能力，目標會接近互動式 Bloomberg / 決策副駕。

整理者判定：新增「數位學徒制 / 專精模型框架」。後續觀察 xAI 是否真的在金融、法律、醫療等高價值領域推出可驗證產品，而不是只把專家名單當敘事。

## 2. Micron：高毛利與五年 SCA，記憶體週期股身分被重估

KP 認為美光財報最驚人的不是 HBM，而是整個記憶體市場被 AI 產能吸走後出現全面缺貨。美光 Non-GAAP 毛利率 74.9%，下一季 guide 到 81%；更反直覺的是，毛利率最高的產品來自 DDR5 與 LP DRAM，而不是 HBM 本身。這意味著 AI server 拉走先進製程後，手機、PC、傳統 server 也同時陷入供應緊張。

真正的框架是 SCA（Strategic Customer Agreement）。過去 memory LTA 多是一年期、約束力弱；美光現在簽下五年、含數量與定價承諾的協議，讓客戶需求和美光 `$25B` capex 綁在一起。若 SCA 具強約束力，這是擴產前先鎖需求；若市場崩盤時仍可重新議價，則可能只是下一輪產能過剩的前奏。

整理者判定：**MU** 既有 watchlist 補強，不建立 `Stocks/MU/`。此段延伸第21期 HBM 價量鎖定 / 記憶體週期鈍化框架，並加入「SCA / 記憶體結構化成長框架」。後續追 SCA 客戶數、duration、pricing floor / flexibility、customer cancellation terms、gross margin、capex、bit supply、DDR5 / LPDRAM pricing、HBM certification 與 Samsung / SK Hynix 供給反應。

## 3. Microsoft / OpenAI / Amazon：AI agent 正在重寫雲端合約

KP 將 Microsoft、OpenAI、Amazon 的衝突拆成一個法律與技術定義問題。Microsoft 依 2025 年底修訂後的合約，主張 Azure 對 OpenAI 的 stateless API 有全球獨家託管權；OpenAI / Amazon 則用 Frontier 的 stateful runtime 切出一個新產品形態：長期記憶、可自主運行、具 agent workflow 的 AI 執行環境。

OpenAI 投向 AWS 的理由是算力與戰略主權。KP 口徑中 Amazon 提供 `$50B` 現金、2GW power 與 Trainium 系列自研晶片，讓 OpenAI 降低 Azure 單點依賴，也接觸「非 AWS 不用」的大型企業客戶。這不是單純多買雲端，而是 OpenAI 嘗試從 Microsoft AI 插件升級為獨立平台。

整理者判定：**AMZN** 為正式已追蹤個股 L3，同步 `Stocks/AMZN/`。這是第31期 AWS / Frontier / Bedrock Stateful Runtime thesis 的後續：若 OpenAI 真能把 stateful agent workload 放到 AWS，AWS 取得的是 enterprise AI runtime 入口；但若 Microsoft 成功要求 toll、功能優先權或 Azure 優先承載，AWS economics 會打折。**MSFT** 為既有 watchlist L3 補強，不建立 `Stocks/MSFT/`。

## 4. OCI 光通訊新憲法：巨頭把規格從供應商手上拿回來

OFC 2026 出現三個關鍵標準：OCI 決定光訊號語言，XPO 定義外部可插拔光模組，Open CPX 規劃未來 co-packaged 光引擎。KP 認為最核心的是 OCI，因為它把光互連從各供應商的專有路線，推向可多供應商替換的 open standard。

OCI 的權力轉移很明顯：創始成員是 **AMD**、**AVGO**、**META**、**MSFT**、**NVDA**、OpenAI，卻沒有傳統光學元件大廠。KP 的判讀是，算力需求方與晶片巨頭要制定規格，讓供應商照規格供貨，降低 vendor lock-in。技術方向上，OCI 用 NRZ、WDM、單纖雙向等規格把光引擎往 GPU / ASIC 更近的位置推，並把約 30W 銅連接功耗壓到約 9W 口徑。

這對 **COHR**、**LITE** 類光學供應商是雙面刃。專有技術溢價下降，但市場從少量客製化模組走向數百萬標準化組件，外部光源、雷射、製造良率和成本控制仍是核心。整理者判定：新增「OCI 光互連標準化 / 矽晶中心框架」，**LITE** watchlist 補強，**COHR** 先進 ticker / theme context，不建立新 `Stocks/`。

## 5. Nvidia 一年一更：全棧擁有與公司晶片組織

KP 從 GTC Q&A 拆 Nvidia 為何能維持 Blackwell 到 Rubin 的一年一更節奏。黃仁勳的答案不是單顆晶片，而是 full-stack ownership：晶片、networking、storage、散熱、機櫃、CUDA 與模型軟體同時設計，運算單位從 GPU 升級成整座資料中心。

組織層面，KP 把 Nvidia 視為一顆公司晶片。黃仁勳有 60 位直接下屬，散熱、網路、晶片、軟體等負責人直接對接核心，減少跨部門整合延遲。這讓 bring-up 不像拼裝系統找 bug，而是硬體第一次通電前，軟體與系統已在模擬中跑過大量驗證。

整理者判定：**NVDA** 既有 watchlist L3 補強。新增「Full-stack ownership / 公司晶片組織框架」，可與 Bytc 的 AI 基建總包工頭框架、大叔 AI Factory 光學互連瓶頸框架交叉比對。後續追 Rubin / Vera / Feynman 節奏、CUDA backward compatibility、NVLink / Spectrum-X、liquid cooling、CPO / optical roadmap、customer deployment cycle 與 gross margin。

## 6. OpenClaw / NemoClaw：Agent OS 與企業安全收編

KP 將 OpenClaw 視為 AI 從 chatbot 進入 agent OS 的標誌：未來使用者不是操作單一 app，而是管理一群可執行任務、寫程式、進系統、互相協作的代理。Nvidia 支持 OpenClaw 的原因，是只要 agent OS 成為標準，長時間在線的 agent 就會持續消耗 dedicated computing。

NemoClaw 是 Nvidia 對 OpenClaw 最大痛點「安全」的補強：sandbox、policy guardrails、privacy routing 讓企業能把原本好玩但危險的 agent project 轉成可控、可部署的 enterprise solution。KP 認為 Nvidia 的策略是用開放標準做大市場，再用安全、性能優化與硬體整合鎖住價值。

整理者判定：**NVDA** watchlist 補強，**MSFT** / **GOOG** / **GOOGL** 只是軟體層競爭 context。新增「OpenClaw / Agent OS 安全收編框架」。後續追 OpenClaw adoption、foundation governance、OpenAI 關係、NemoClaw enterprise adoption、OpenShell runtime、Nemotron attach、RTX / DGX optimization、agent security incidents 與 dedicated computing demand。
