## 總結

第35期的主軸是「AI 泡沫敘事進入現金流、算力結構與法規現實檢查」。KP 開場先提醒，市場波動極端時不要把短線投機誤認成價值投資；投資人要先誠實檢查組合，降低太重、不了解或純投機的部位，先建立不會在風暴中被摧毀的防線。

本期正式同步已追蹤個股的是 **AMZN**、**INTC**、**LRCX**、**GOOG** / **GOOGL**。**AMZN** 段落把 AWS 內部 sales / technical-support agent 與 Bedrock marketplace 連成 enterprise AI agent factory，直接延伸 AWS enterprise AI OS thesis。**INTC** 段落不是單純看多 CPU，而是將 Agentic AI CPU 復權與 Arm 直接造晶片放在一起：CPU TAM 擴張，但 x86 share 風險同步升高。**LRCX** 段落來自 Musk Terafab 的 WFE / deposition / etch 收費站 read-through。**GOOGL** 段落來自 Google TurboQuant，重點是 AI efficiency 可能短期打擊 memory stock 情緒，但長期透過 Jevons paradox 放大 AI workload。

OpenAI / Sora 與 CLARITY Act 兩段不建立正式 `Stocks/` 專案；前者補「AI 煙火 vs 印鈔機」框架，後者補 stablecoin 支付軌道與數位儲蓄帳戶的分流。**ARM**、**ASML**、**AMAT**、**KLAC**、**MU**、**CRCL**、**COIN** 等更新 ticker / watchlist / framework。

## 0. 風暴前組合體檢：先確保不會被摧毀

KP 開場將極端波動下的第一優先順序放在防守：投資人要把短期投機、價值投資和情緒交易分開，誠實檢查部位是否太重、是否真的理解、是否只是因為 FOMO 或不願認錯而持有。關掉 app 之前，先問組合在更糟糕的市場裡會不會被摧毀。

整理者判定：這段新增「風暴前組合體檢 / 防禦部署框架」。它不是看空或叫人清倉，而是把研究流程前置到風險承受力：如果一個部位在 thesis 還沒被驗證前就足以毀掉組合，那不是投資問題，而是 sizing / liquidity / conviction 問題。

## 1. Amazon：Bedrock agent factory 與 SaaS 背景板

KP 記錄 Amazon / AWS 先把 AI agent 用在內部 sales 和 technical support：銷售人員可透過基於 Amazon Bedrock 的 agent，調用內部技術知識回答客戶問題，減少對人類技術專家的即時依賴。這段的重點不是 Amazon 正在賣某個單一 agent，而是它先用自己作為測試場，證明 agent 可以取代部分昂貴流程。

對外部客戶，Amazon 賣的是「agent factory」：Bedrock、企業資料、permissions、workflow、Marketplace agents and tools。KP 認為企業軟體正在從「人操作工具」轉向「AI 在背景執行工作」。若公司未來直接雇用 AI agents，而不是替更多人買 SaaS seats，傳統 seat-based SaaS valuation 就會繼續承壓。KP 用 IGV 年初至今跌逾 20% 作為 SaaS de-rating 的溫度計，提醒不是所有下跌都值得接刀，要分辨誰只是裸泳、誰真的能演化。

整理者判定：**AMZN** 正式已追蹤個股 L3，新增 / 補強「Bedrock 造代理工廠 / SaaS 背景板框架」。這與第31期 Stateful Runtime、第34期 OpenAI / AWS stateful runtime 合約邊界相接：AWS 若能把 Bedrock agent factory 變成企業工作流執行層，高 capex 與 platform strategy 更容易被視為有回收路徑。風險是 agent revenue、Bedrock margin、customer ROI、data security 與 SaaS de-rating 不必然全流向 AWS。

## 2. Arm：Agentic AI CPU 復權與中立 IP 商模式被重寫

KP 將 Rene Haas 的 CPU demand comment 放進 Agentic AI 架構變化中：GPU 是肌肉，CPU 則像 cortex，負責 logic scheduling、environment awareness、autonomous decisions 和多步工具調度。AI agent 在一兩分鐘內可產生遠超人類的 token / action volume，而這些 orchestration、queueing、routing、system-state tracking 都大量依賴 CPU。

真正改變是 Arm 直接下場造晶片。Arm 過去像半導體瑞士，賣 IP 給 Qualcomm、Nvidia、hyperscalers 與其他客戶；現在推出自家 **Arm AGI CPU**，KP 口徑為 136 cores、TSMC 3nm、針對 AI agent workloads、energy efficiency 約為競品 2x。這讓 Arm 有機會從 royalty / IP layer 走向更高 ASP / 更高控制權，但也可能和既有 IP 客戶發生 channel conflict。

對 **INTC** 的含義是雙面：Agentic AI 讓 CPU TAM 擴大，支持大叔 / Bytc 近期 CPU 復權框架；但 Arm、Nvidia CPU-only rack、Meta meaningful pure CPU data-center order 也提醒，增量 CPU 需求不等於 Xeon 自動受益。整理者判定 **INTC** 為 L2-L3：補強 CPU 復權的需求端，也提高 Arm / Nvidia / AMD / custom CPU 對 x86 share 的競爭風險。

## 3. Musk Terafab：先問誰收製程與設備的過路費

KP 對 Musk Terafab 的處理不是先判斷它能不能最終成功，而是把投資問題拆成兩層：第一層是終局風險，包括垂直整合、2nm 良率、人才、資本與供應鏈是否可行；第二層是過程確定性，也就是在嘗試之前，誰必須先被採購。

如果 Terafab 真的要追求 1 Terawatt AI compute、約 50x current global output 的願景，WFE capex 必然先行。KP 把 **ASML** 寫成 EUV / High-NA EUV gatekeeper，單機約 `$300M-$400M`；**KLAC** 是 inspection / metrology 品保總監；**AMAT** 與 **LRCX** 則是 deposition / etch 建築工頭。Terafab 不論最後誰贏，只要要蓋先進半導體產能，設備商先收費。

整理者判定：**LRCX** 正式已追蹤個股 L2 補強，建立 2026Q1 季度筆記。這不是 Lam-specific 財報或訂單 thesis，而是 WFE process intensity / capex optionality 的補充。對 watchlist，更新 **ASML**、**AMAT**，新增 **KLAC**。另 KP 提到 Terafab 可能需要 >10GW power，讓 Texas grid 與 stable baseload / nuclear / power suppliers 具有議價權；但本篇未具名形成正式電力股 thesis。

## 4. Google TurboQuant：記憶體牆、短期交易理性與長期 Jevons

Google TurboQuant 是本期最容易被誤讀的一段。KP 記錄 TurboQuant 可將 KV cache memory 壓縮至少 6x，部分 compute speed +8%，不需 retraining，且 hardware agnostic。市場短期把這解讀成 memory demand 下降，記憶體股賣壓加劇。KP 認為短線交易上這有合理性：memory stocks 前期漲太多、部位擁擠，遇到 headline risk 去槓桿並不奇怪。

但結構上，KP 判斷是 Jevons paradox。降低每單位推理 / 長上下文 / agent memory 成本，可能不是讓 memory demand 消失，而是讓更多應用突然變得可行，最終把總需求推高。KP 用 DeepSeek V3 / R1 的 MLA 作對照：2025 年 1 月 MLA 一度讓 KV cache memory -93%、memory use 20-60x less，初期也引發 panic；但 deployment explosion 之後，HBM / memory demand 反而上升。

整理者判定：**GOOG** / **GOOGL** 為 L2 Stocks 補強。這段不直接把 TurboQuant 寫成 Google 單股 L3 monetization，但它強化 Google 在 AI infra software efficiency 的能力：若能降低 cost per task，Cloud / Search / agent inference 的 economics 會改善。**MU** 與 memory makers則更新 watchlist / ticker，標記「短期交易理性 vs 長期結構理性」與「記憶體牆 Jevons 框架」。

## 5. OpenAI Sora：AI 產業從煙火回到印鈔機

KP 將 OpenAI 砍 Sora、instant checkout、adult mode，甚至可能放慢 Atlas browser / Health 等方向，解讀為 AI sector 進入 Focus Era。Sora 是典型 fireworks：聲量大、demo 性感、Disney licensing 有故事，但每日 compute cost 可超過 `$10M`，lifetime IAP 只有幾百萬美元量級，無法支撐資源配置。

對比是 Anthropic Claude Code 類 enterprise / developer workflow 正在拿走更清楚的付費需求。KP 的結論是，AI 公司正在從「什麼都要做」轉向「B2B、developer tools、workflow、長期 AGI infrastructure」這些更接近印鈔機的位置。OpenAI 未上市，因此本段不建 ticker；**MSFT** 作為 OpenAI exposure 與 enterprise AI monetization context，**DIS** 作為早期 AI video IP licensing context。

整理者判定：新增「AI 煙火 vs 印鈔機 / Focus Era 框架」。它可用來檢查所有 AI app：吸引注意力不等於可持續收入，產品若無法把 compute cost、retention、enterprise willingness-to-pay 與 gross margin 串起來，就容易在資源重新分配時被砍。

## 6. CLARITY Act：stablecoin 支付軌道，不是數位高息儲蓄帳戶

KP 將 CLARITY Act draft 的 passive yield ban 拆成「禁止穩定幣發行方付利息或功能等同利息的被動收益」，而不是禁止所有 DeFi / lending / activity-based yield。被打擊的是把 USDC / stablecoin 包裝成類 HYSA / money-market substitute 的敘事。

對 **CRCL**，KP 認為 Circle 的核心 reserve-yield-from-circulation thesis 沒被直接摧毀。Circle USDC reserve 約 `$75B-$80B`，FY2025 reserve interest 約 `$2.64B`，佔 revenue 95%-96%；CLARITY 若限制對持有人 pass-through yield，可能降低 stablecoin 作為被動儲蓄帳戶的吸引力，但 USDC 的主要用戶仍為 crypto liquidity、24/7 transfer、DeFi / payments / settlement rails。KP 的判斷是 growth slows but does not reverse，payment rails thesis 仍在。

對 **COIN**，風險更直接。Coinbase on-platform USDC 約 `$17.8B`、約 24% circulation；Coinbase 依 on-platform 100%、off-platform 50% revenue-share 從 Circle 收入並提供 rewards。若 passive yield 受限，Coinbase 用 USDC rewards 吸引 lazy money 的策略更受挑戰。整理者判定：**CRCL** 更新既有 watchlist，**COIN** 新增 watchlist；不建立正式 `Stocks/`。
