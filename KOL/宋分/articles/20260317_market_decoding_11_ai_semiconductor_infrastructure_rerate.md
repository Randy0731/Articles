## 主旨

這篇是宋分在 2026-03-16 備忘錄 #2 之後，把 HBM / 記憶體雙結構問題推向更大的 AI 半導體估值框架：市場過去問的是「今年手機會賣幾支」，現在可能開始問「AI infrastructure 未來十年會蓋多大」。如果 AI 把半導體需求從消費電子 cycle 轉成 infrastructure cycle，市場就可能不再只用景氣循環框架評價整條半導體鏈。

## HBM：不只是 memory component，而是 AI system component

宋分延續備忘錄 #2 的問題：如果記憶體產業真的出現雙結構，AI 產品部分跟著 CSP CAPEX cycle 走，記憶體會不會像 CSP 一樣在第一階段與第二階段之間來回，而不是很快走向第三階段的成長放緩。

他的新增拆解是：傳統 DRAM 高度同質化，客戶可在供應商之間切換，因此過去循環常是需求上升、價格上漲、廠商擴產、供給過剩、價格回落。但 HBM 有幾個不同特徵：ASP 高於傳統 DRAM、技術門檻與良率要求高、需要先進封裝，而且需求幾乎完全來自 AI GPU / AI server。

這使 HBM 可能從單純記憶體元件，變成 AI 系統的一部分。若 HBM 必須與 GPU、先進封裝與系統設計深度協作，供應鏈綁定可能變強；若不同廠商在堆疊技術、良率、功耗與頻寬上拉開差距，產品差異化也可能上升。

但宋分保留反面：高利潤會吸引資本與產能。若記憶體廠與封裝廠加速投資，HBM 是否能長期維持高利潤與供給限制仍是開放問題。整理者判斷：本篇不是把 HBM 寫成永久高毛利，而是把觀察焦點從「整體記憶體 cycle」拆成「AI memory 子市場是否被單獨 re-rate」。

## 從台積電到 AI 半導體：問題被改寫時，估值框架也被改寫

宋分用台積電作為半導體 valuation framework 改變的歷史類比。台積電也曾被市場視為景氣循環股，需求高度依賴 PC / 消費電子；當製程優勢拉開、Apple 等大型客戶需求變成更結構性的來源後，市場開始把它從 cyclical semiconductor manufacturer 重新定義為 structural technology infrastructure provider。

這段類比的真正問題是：如果 AI 創造新的需求結構，市場會不會開始只對「AI 那一部分」重新定價。對記憶體來說，問題可能從「整家公司是不是 cyclical commodity producer」轉為「哪家公司在 AI memory 結構裡佔據重要位置」。對更廣泛半導體來說，問題可能從「這個 semiconductor cycle 什麼時候結束」轉為「AI infrastructure 的長期規模到底多大」。

## MSFT、AMZN、COST：長期結構需求如何反覆重啟第一階段

宋分用三個案例說明有些公司不是沒有 cycle，而是 cycle 結構不同。

| 公司 / 類型 | 第一階段重啟邏輯 | 第二階段 / 驗證點 | 本文用途 |
|---|---|---|---|
| **MSFT** | Windows / Office 是 PC 基礎設施；後來 Office SaaS 化與 Azure 讓 MSFT 重新成為 enterprise IT infrastructure；AI 又把問題改成 enterprise AI infrastructure | 目前市場等待 Copilot / Azure AI / OpenAI 相關貨幣化讓基本面追上估值 | 說明偉大公司可因新敘事多次回到第一階段 |
| **AMZN** | 電商規模飛輪先累積現金流，再用 AWS 變成 cloud infrastructure，現在又把 AWS 推向 AI infrastructure | 目前市場等待 AI CAPEX / AWS AI monetization 交成績 | 補強 AMZN 從零售變 cloud + retail platform，再到 AI infrastructure 的多次重定價 |
| **COST** | 會員費與高續約率讓現金流可預測，市場買的是會員數量的長期成長 | 開新店、進新國家、增加會員 | 說明可預測性本身可形成高估值，不一定要靠技術革命 |

這三個案例的共通點是：市場相信需求來自長期結構，而不是短期景氣。它們的 cycle 不是需求爆發後供給擴張、週期結束，而是新敘事出現、估值擴張，基本面驗證後成長延續，新的敘事再出現、再次定價。

## AI 半導體的三階段與下一批敘事

宋分把 AI 半導體可能的 re-rate 分成三段：

| 階段 | 市場問題 | 典型訊號 |
|---|---|---|
| 第一階段：市場開始相信 AI 需求是真的 | 這會不會只是下一個 semiconductor boom？ | GPU 需求好、記憶體價格上升、資料中心 CAPEX 強 |
| 第二階段：市場開始重新定義產業 | AI infrastructure 的長期規模到底多大？ | 雲端公司持續擴建資料中心，大型模型持續需要算力，需求沒有隨景氣衰退消失 |
| 第三階段：新敘事再次出現 | 下一個能讓產業回到第一階段的敘事是什麼？ | AI training cluster、AI inference infrastructure、edge AI、AI networking 等新技術浪潮 |

整理者判斷：這段補強 2026-03-07 的定價三階段。對典型週期股，第三階段是 multiple compression；但若 AI 半導體被市場重新定義為 infrastructure，新的需求敘事可能讓它反覆回到第一階段與第二階段，而不是線性走向週期結束。

## 2026-03-20 補充 Note：不斷重啟新的定價

2026-03-20 的《產業與個股透鏡 #3｜技術與商業模式》是同一篇 3/17 市場解碼深度文的補充推廣 Note，source_id 記為 `宋分-20260320-pricing-cycle-restart-msft-amzn-cost-9dd0938f`，並以本篇作 duplicate_of，不另開重複長文。

這篇補充把原本的基礎設施敘事循環講得更直觀：能穿越牛熊的公司不是沒有 cycle，而是每次市場用新框架看它時，會重新進入「新敘事 -> 估值擴張 -> 基本面驗證 -> 再一個新敘事 -> 再定價」的路徑。**MSFT** 的 PC / cloud / AI infrastructure、**AMZN** 的電商現金流 / AWS / AI infrastructure、**COST** 的會員現金流，都被用來說明「結構性需求」如何讓市場不只用傳統週期高低來定價。

對 AI 半導體的補充語句是：如果 AI infrastructure 真的是長期結構，市場問題就可能從「這是不是下一個 semiconductor cycle」轉為「AI infrastructure 到底會有多大」。這句話補強本文既有結論：AI 半導體仍在第一階段，GPU、記憶體與 CAPEX 已強，但後續能否穿越牛熊，要看基礎設施需求、收入可預測性與 AI revenue mix 能否被基本面驗證。

## 機構 re-rate 產業的五條件

| 條件 | 宋分定義 | 追蹤意義 |
|---|---|---|
| 收入更可預測 | 公司開始賺更高品質的錢，例如一次性收入變訂閱收入，或消費電子變 infrastructure | 決定市場是否願意降低折現率、提高 multiple |
| 現金流更穩定 | 市場不是只為成長付錢，而是為可預測性付錢 | 對照 **COST**、CSP、AI infra suppliers |
| 公司成為產業基礎設施 | 整個產業都需要你的產品，估值會完全不同 | 對照 **NVDA** / **ASML**、CUDA / tool ecosystem |
| 產業競爭減少 | 例如記憶體從多家公司變三國鼎立，市場相信價格戰較少 | 對 memory / HBM oligopoly 與 capex discipline 重要 |
| 管理層資本配置被信任 | 不亂擴產、回購、控制費用，讓市場願意提高信任 | 對重資本半導體特別重要，因供給錯配會毀掉 cycle |

## AI 半導體六個追蹤指標

| 指標 | 要回答的問題 | 可能支持 re-rate 的訊號 | 反面風險 |
|---|---|---|---|
| AI CAPEX | 科技巨頭願意花多少錢買 AI 硬體 | CAPEX 持續、但有 revenue / ROI 驗證 | overbuild、FCF 壓力、capex 見頂 |
| AI semiconductor intensity | 一個 AI data center 裡需要多少半導體 | 每個 AI system 的 GPU / HBM / networking / storage 含量提高 | 系統優化降低 chip / memory intensity |
| AI cluster scale | 模型越大是否需要更多 chip | cluster 持續放大，training / inference 同時擴張 | 模型效率提升或需求集中度下降 |
| AI hardware refresh cycle | AI 技術進步是否讓硬體每 3-4 年更新 | 類似手機週期，持續形成新需求 | refresh 週期拉長或二手市場壓價 |
| AI ecosystem lock-in | 誰控制 AI 工具生態系 | CUDA / software stack / integration 讓替代困難 | 開源、ASIC、CSP 自研降低 lock-in |
| AI infrastructure revenue mix | AI revenue 是否足以 cover non-AI business | AI 營收比重提高且 margin 可維持 | 非 AI 業務下滑、AI margin 低於預期 |
