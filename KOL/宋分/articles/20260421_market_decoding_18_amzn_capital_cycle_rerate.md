宋分這篇把 Amazon 從「成長股」重新定義為一台「現金流 -> 再投資 -> 再爆發」的資本週期機器。文章主張 Amazon 長期盤整不是因為缺乏成長，而是市場在每一輪重資本投入期都先用錯框架看它：短期 EPS / FCF 難看，背後可能是在把當期費用轉成下一輪現金流引擎。

## 宋分框架拆解

### 1. Amazon 不是單純成長股，而是資本週期股

宋分認為 Amazon 的歷史節奏不是穩定線性成長，而是先大幅投入、壓低利潤，等市場看見現金流與利潤釋放後再重新定價。這也是為什麼 Amazon 常出現多年盤整後，一旦啟動就走出多年趨勢的型態。

整理者判斷，這段把 2/6 的「AWS Cycle 3 建置期」從單次財報解讀擴成更一般的資本週期框架：市場不一定討厭 CAPEX，而是討厭看不懂 CAPEX 最後會不會變成可預測現金流。

### 2. 三個 Cycle：電商 / AWS / AI + 衛星

文章將 Amazon 歷史拆成三輪：

| Cycle | 市場當時的誤判 | 宋分的解讀 | 追蹤重點 |
|---|---|---|---|
| Cycle 1：電商 / 物流 | 市場懷疑 Amazon 是否能活下來 | 物流與規模先壓低利潤，後來形成零售現金流與平台能力 | retail revenue、shipping cost、fulfillment efficiency、3P services |
| Cycle 2：AWS | 市場把 AWS 當副業，仍用低毛利零售看 Amazon | AWS 成為高毛利利潤引擎，讓市場第一次大幅重估 | AWS revenue growth、AWS operating margin、backlog / RPO |
| Cycle 3：AI + 衛星 | 市場看到 CAPEX、EPS 壓力與遠期衛星不確定性 | Amazon 可能用 AWS / 平台現金流投向 AI、連網能力與更底層基礎設施 | AI demand、Trainium / Bedrock、Amazon Leo / D2D、ROIC turn |

### 3. 會計會把未來護城河藏在當期費用

宋分反覆提醒，Amazon 很多支出在會計上看起來像費用，但本質可能是在建立未來現金流，例如物流網路、Prime 補貼、AWS 初期資料中心。若市場只看當季 EPS，容易把這類投資誤判成永久性的低利潤。

整理者判斷，這段與既有「資本黑洞 / 複利成長分流框架」相接：不是所有高 CAPEX 或低 EPS 都值得原諒，關鍵要看這些支出是否提高未來需求鎖定、現金流可見度與資本效率。

### 4. 如何追蹤 Amazon 是否在下一輪爆發軌道上

宋分把追蹤拆成三塊：

| 面向 | 宋分追蹤方式 | 整理者判斷 |
|---|---|---|
| 零售 | 看實體店面、線上零售、第三方賣家服務營收是否穩定，以及運費 / fulfillment cost 是否被壓下來 | 零售不一定是利潤主體，但提供流量、規模與現金流底座 |
| AWS | 看 AWS revenue、backlog / revenue 動能、operating margin 是否仍能貢獻主要利潤 | 承接 2/6 的 AWS backlog 先行框架；需用 filings / call 持續校準 |
| 廣告 + Prime | 看高毛利收入占比是否提高、Prime 是否有漲價與 ARPU 空間 | 這是 consolidated margin 改善的關鍵，不應只看低毛利零售營收占比 |

### 5. 衛星是否能讓 Amazon 再次 re-rate

宋分的重點不是衛星本身立刻賺多少錢，而是 Amazon 是否正在掌握「連網能力」這個更底層資源。若 Globalstar / Amazon Leo D2D 變成 consumer、enterprise、government 的基礎連接層，市場可能將其從遠期燒錢項目改看成新現金流可見度。

Amazon official 於 2026-04-14 宣布與 Globalstar 簽訂 definitive merger agreement，目標是把 Globalstar 衛星、頻譜與營運能力併入 Amazon Leo，增加 direct-to-device 服務；官方也提到與 Apple 的衛星服務協議、交易預期 2027 年完成且仍需監管批准與 HIBLEO-4 replacement satellite milestones。這些官方資訊支持「事件存在」，但不等於現金流已兌現。

### 6. ROIC x Organic Growth 才是股價爆發條件

宋分把核心公式寫成 ROIC x Organic Growth：ROIC 代表資本效率，Organic Growth 代表成長來源。只有高 ROIC 沒有成長，通常只是穩定估值；只有成長沒有 ROIC，則可能變成資本黑洞。Amazon 難懂之處在於它常故意用 CAPEX 壓低短期 ROIC，等新現金流引擎啟動後 ROIC 才突然上升。

整理者判斷，後續看 **AMZN** 不應只問「CAPEX 高不高」，而要問：ROIC 是否開始轉折？轉折時 organic growth 還在不在？如果二者同時成立，才比較接近宋分所說的下一段 re-rate。

### 7. 如何用 Amazon 框架找下一個大牛股

文章最後把 Amazon 抽象成四個篩選條件：

| 條件 | 判斷問題 |
|---|---|
| 現金流是否有雙層結構 | 低毛利或高投入業務是否提供規模 / 流量，高毛利層是否負責利潤釋放 |
| 市場是否用錯估值模型 | 市場是否仍把公司當舊分類，例如零售、工具或硬體，而不是新基礎設施 |
| CAPEX 是防守還是進攻 | 資本支出只是維持既有業務，還是在買下一層需求鎖定 |
| 費用背後是成本還是投資 | 當期費用是否建立未來可預測現金流，而不是單純營運效率惡化 |

本篇應與 2026-02-06 AMZN CAPEX Cycle 3 一起讀：2/6 是財報後估值與 backlog 先行；4/21 是完整公司分類與資本週期框架。若查詢 **AMZN**，這篇會成為正式 `Stocks/AMZN/` 專案的核心底稿之一。
