## 主旨

宋分把 Amazon 財報後的 CAPEX 恐慌，類比成 AWS 前兩輪建置週期：先大幅投資、現金流和 EPS 被壓，後續才讓 AWS 營收、營益率與 EPS 跳升。他認為市場只看見 CAPEX 暴增與自由現金流轉弱，但忽略 AWS backlog YoY 在 2025Q3-Q4 從 `22%` 跳到 `38%`，因此把 **AMZN** 視為與 **NVDA** 並列的 M7 最便宜標的之一。

## 宋分框架拆解

### CAPEX 建置週期 / AWS backlog 先行框架

**核心邏輯**：當市場因 CAPEX 暴增、FCF 轉弱或 EPS 被壓而恐慌時，宋分回看 AWS 的歷史建置週期：2016-2019 建置帶來 2021 AWS 營收與 EPS 跳升，2022 Cycle 2 壓低現金流與 EPS，2023-2024 又反映到營益率與 EPS。這次他把 2026 前後的 AI / AWS capacity 投資視為 Cycle 3，而 AWS backlog YoY 加速是判斷 CAPEX 有需求承接的先行訊號。

**適用情境**：hyperscaler 或 AI infrastructure 公司在重資本支出期被市場懲罰，但 backlog / RPO / customer commitments 已領先收入加速時。

**觀察指標**：CAPEX / revenue、FCF margin、AWS revenue YoY、backlog / RPO YoY、AWS operating margin、EPS revision、customer commitments、AI / chips / cloud capacity utilization、management CAPEX guide。

### 評價溫度計 / M7 相對便宜

**核心邏輯**：宋分把 **AMZN** 放進同一套評價溫度計：歷史評價區間、成長確定性、現金流穩定度與市場預期位置。因 forward PE 壓到 `21x`，且市場焦點集中在 CAPEX 與 FCF 壓力，他認為 **AMZN** 已接近左側，不是天花板附近。
